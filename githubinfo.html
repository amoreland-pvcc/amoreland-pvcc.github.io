All About Github

  1. Q. What is a version control system?
  2. A. Version control, also known as source control, is the practice of tracking and managing changes to software code. Version control systems are software tools that help software teams manage changes to source code over time.
  3. Q. What is Git?
  4. A. Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
  5. Q. What is Github?
  6. A. GitHub is a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere.
  7. Q. What is the difference between Git and Github?
  8. A. While Git is a tool that's used to manage multiple versions of source code edits that are then transferred to files in a Git repository, GitHub serves as a location for uploading copies of a Git repository.
  9. Q. Who started Github and how was it started?
  10. A. Wanstrath and Preston-Werner needed GitHub for their own work, and so they developed their tool out of necessity. The two men quickly identified a major problem in their work—forking code branches and collaborating on programming projects—and devised a solution that met their needs. What was brilliant about Wanstrath and Preston-Werner’s solution is that every software developer, regardless of programming language or operating system or job role, experienced these major problems. This represented an immense potential market for their future product.
  11. Q. What company owns it now?
  12. A. Microsoft

Git/Github terms:

  1. repository - a central location in which data is stored and managed
  2. commit - the updating of a record in a database
  3. fork - to take the source code from an open source software program and develop an entirely new program
  4. push - the delivery of information on the Web that is initiated by the information server rather than by the information user or client
  5. pull requests - an event that takes place in software development when a contributor/developer is ready to begin the process of merging new code changes with the main project repository
  6. workflows - a sequence of tasks that processes a set of data through a specific path form intitiation to completion
  7. issues - a unit of work to accomplish an improvement in a data system
  8. raw button - the button in Github that allows users to view the contents of a file in its raw, unrendered form
  9. blame button - a feature on Github that allows users to view the revision history of a file and see who made each change to the file and when
